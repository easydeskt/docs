Входящие обращения поступают через личные сообщения сообщества VK. Убедитесь, что в настройках сообщества включены личные сообщения.

## Токен сообщества

1. Перейти в сообщество → **Управление** → **Работа с API** → **Ключи доступа**.
2. Создать ключ с правами на **Сообщения**.
3. Скопировать токен.

## Код подтверждения (только для Callback API)

Callback API требует подтвердить, что сервер принадлежит вам. VK пришлёт строку проверки на ваш endpoint при настройке сервера в интерфейсе VK — её нужно вернуть в ответе.

Код подтверждения находится в настройках сообщества → **Работа с API** → **Callback API** → вкладка **Настройки сервера**.

## Переменные в .env

```env
CHANNEL_VK_COMMUNITY_TOKEN=токен-сообщества
CHANNEL_VK_CALLBACK_SECRET=секрет-для-верификации-запросов
CHANNEL_VK_CONFIRMATION_CODE=строка-подтверждения-из-vk
```

`CHANNEL_VK_CALLBACK_SECRET` — произвольная строка. Задаётся в настройках сервера в интерфейсе VK и в `.env` — должна совпадать.

## Режимы работы

| Режим | Описание |
|-------|----------|
| Callback API | VK отправляет события POST-запросом на ваш URL. Требует HTTPS и публичного домена. |
| Long Poll | Сервер сам опрашивает VK API. Работает без публичного домена. |

:::tip
Для локальной разработки используйте Long Poll — не нужен публичный домен. Для продакшена предпочтительнее Callback API: меньше задержка доставки событий.
:::
