# Жизненный цикл тикета

:::note[Раздел в разработке]
Полное описание с диаграммами переходов готовится.
:::

## Статусы

| Статус | Описание |
|--------|----------|
| `OPEN` | Тикет создан, оператор не назначен |
| `IN_PROGRESS` | Назначен на оператора, работа ведётся |
| `RESOLVED` | Вопрос решён (`/resolve`) |
| `CLOSED` | Тикет закрыт (`/close`) |
| `MERGED` | Объединён с другим тикетом (`/merge`) |

## Переходы

- `OPEN` → `IN_PROGRESS` — при назначении оператора (`/assign`)
- `IN_PROGRESS` → `OPEN` — при снятии назначения (`/free`) или перехвате другим оператором
- `IN_PROGRESS` → `RESOLVED` — командой `/resolve`
- `RESOLVED` → `IN_PROGRESS` — автоматически, если клиент написал новое сообщение
- Любой → `CLOSED` — командой `/close`
- Любой → `MERGED` — командой `/merge #id`

:::caution
Тикет в статусе `MERGED` недоступен для дальнейшей работы. Все входящие сообщения от клиента перенаправляются в целевой тикет.
:::
