## Статусы

| Статус | Описание |
|--------|----------|
| `OPEN` | Тикет создан, оператор не назначен |
| `IN_PROGRESS` | Назначен на оператора, работа ведётся |
| `RESOLVED` | Вопрос решён (`/resolve`) |
| `CLOSED` | Тикет закрыт (`/close`) |
| `MERGED` | Объединён с другим тикетом (`/merge`) |

## Переходы

**Открытие и назначение**
- Новый тикет всегда создаётся со статусом `OPEN`.
- `/assign` → `IN_PROGRESS`. Тикет назначен, работа началась.
- `/free` → `OPEN`. Назначение снято, тикет снова доступен для взятия.

**Решение**
- `/resolve` → `RESOLVED`. Вопрос решён.
- Если клиент напишет ещё раз после `/resolve` — тикет автоматически возвращается в `IN_PROGRESS`. Новый топик не создаётся, переписка продолжается в том же.

**Закрытие**
- `/close` переводит тикет в `CLOSED` из любого статуса. Используется для спама, дублей, отозванных обращений.
- `/merge #id` переводит тикет в `MERGED` из любого статуса. Все входящие от клиента перенаправляются в указанный тикет.

:::caution
Тикет в статусе `MERGED` нельзя переоткрыть или переназначить. Если нужно разделить объединённые тикеты обратно — придётся создавать новый вручную.
:::

## Когда использовать `/resolve` vs `/close`

`/resolve` — клиент получил ответ, вопрос закрыт. Если он напишет снова по тому же вопросу, тикет автоматически возобновится.

`/close` — тикет не должен возобновляться. Спам, ошибочное обращение, дубль уже объединённого тикета.
