## Приоритеты

Устанавливаются командой `/priority` прямо в топике:

| Приоритет | Команда | Когда использовать |
|-----------|---------|-------------------|
| Низкий | `/priority low` | Общие вопросы без срока, информационные запросы |
| Обычный | `/priority normal` | Стандартное обращение (по умолчанию) |
| Высокий | `/priority high` | Проблема влияет на работу клиента |
| Срочный | `/priority urgent` | Сервис недоступен, критическая ошибка, потеря данных |

В Mini App тикеты сортируются с учётом приоритета: срочные — всегда наверху списка.

Приоритет можно поменять в любой момент: сначала взять тикет в работу необязательно.

## Теги

Теги — произвольные метки для категоризации тикетов. Набор тегов настраивает администратор, оператор назначает их через карточку тикета в Mini App.

Один тикет может иметь несколько тегов. В списке тикетов можно фильтровать по одному или нескольким тегам одновременно.

**Типичные теги:** `billing`, `bug`, `feature-request`, `vip`, `refund`, `docs`

:::tip
Приоритет и теги решают разные задачи. Приоритет — про срочность. Теги — про категорию обращения. Срочный баг можно пометить одновременно `/priority urgent` и тегом `bug`.
:::
