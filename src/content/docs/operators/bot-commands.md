---
title: Команды бота
description: Полный справочник команд Telegram-бота для операторов
---

Все команды вводятся внутри топика тикета в супергруппе.

## Назначение

| Команда | Описание |
|---------|----------|
| `/assign me` | Назначить тикет на себя |
| `/assign @username` | Назначить тикет на другого оператора по его Telegram-юзернейму |
| `/free` | Снять назначение — тикет переходит обратно в статус `OPEN` |

`/assign me` работает даже если тикет уже назначен на кого-то другого. Это перехват: система не блокирует передачу, достаточно договориться с коллегой.

## Переписка с клиентом

| Команда | Описание |
|---------|----------|
| `/reply Текст` | Отправить ответ клиенту в канал, откуда пришло обращение |

Текст пишется прямо после команды: `/reply Спасибо за обращение, уточните номер заказа.`

Если нужно отправить файл или изображение — прикрепите его к сообщению с командой `/reply` как вложение. Вложение дойдёт клиенту вместе с текстом.

## Управление статусом

| Команда | Описание |
|---------|----------|
| `/resolve` | Отметить тикет как решённый. Если клиент напишет ещё раз — тикет автоматически возобновится (`IN_PROGRESS`) |
| `/close` | Закрыть тикет без решения. Используется для спама, дублей, ошибочных обращений |
| `/merge #id` | Объединить с тикетом `#id`. Все последующие сообщения от клиента придут в целевой тикет |

## Приоритет

| Команда | Описание |
|---------|----------|
| `/priority low` | Низкий приоритет |
| `/priority normal` | Обычный (по умолчанию) |
| `/priority high` | Высокий |
| `/priority urgent` | Срочный |

В Mini App тикеты с приоритетом `urgent` отображаются первыми в списке.

## Заметки

| Команда | Описание |
|---------|----------|
| `/note Текст` | Добавить внутреннюю заметку. Не видна клиенту, видна всем операторам в карточке тикета в Mini App |
| `/notes` | Показать все заметки о клиенте этого тикета. Заметки привязаны к клиенту, а не к конкретному тикету — они видны во всех его обращениях |

:::tip
Заметки удобно использовать для контекста: VIP-клиент, активный договор, прошлые проблемы. Один раз записал — видно во всех следующих тикетах этого клиента.
:::
