# Обзор рабочего процесса

:::note[Раздел в разработке]
Полная документация пишется. Ниже — краткая схема рабочего процесса.
:::

Клиент отправляет сообщение через один из подключённых каналов (Telegram, Email, ВКонтакте). EasyDesk создаёт **тикет** и открывает для него отдельный топик в Telegram-суперагруппе — это основное рабочее место оператора.

**Базовый сценарий:**

1. Клиент пишет в Telegram-бот, на email или в сообщения сообщества VK
2. В суперагруппе появляется новый топик с заголовком по шаблону
3. Оператор назначает тикет на себя командой `/assign me`
4. Ответ отправляется клиенту командой `/reply Текст`
5. После решения вопроса — `/resolve` или `/close`

**Особенности:**

- Один тикет = один топик. История переписки видна прямо в теме.
- Перехват: любой оператор может взять незакрытый тикет командой `/assign me` без согласования.
- Параллельно можно работать через **Mini App** — там доступен список тикетов с фильтрами и полная карточка.
